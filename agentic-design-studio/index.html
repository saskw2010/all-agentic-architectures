من التصميم إلى الكود

الحقيقة غير المريحة: استخراج HTML وCSS لا يصنع تطبيقًا Production-Ready. القيمة الحقيقية هي تحويل التصميم إلى Components ملتزمة بالـDesign System، الصلاحيات، Accessibility، الـState، والـBusiness Rules.

WHY THIS MATTERS

لماذا يدخل Penpot في معمارية Sky365؟

نستخدم Penpot كـDesign Engine مفتوح وقابل للاستضافة الذاتية، وليس كهوية المنتج. هذا يحافظ على استقلال المعمارية ويسمح بإضافة أدوات أخرى مستقبلًا.

01 · OPEN FOUNDATION

ملكية البيانات والبنية

إمكانية الاستضافة الذاتية تجعل ملفات التصميم والـLibraries والـTokens داخل البنية التي نتحكم فيها، مع خيارات Docker وKubernetes الرسمية.

02 · DESIGN ↔ CODE

جسر حقيقي لا تسليم يدوي

الـMCP يستطيع قراءة بنية الملف، المكونات، الـStyles والـTokens، ثم دعم Design-to-Code وCode-to-Design ضمن Workflow منضبط.

03 · DESIGN SYSTEM

Design Tokens كعقد مشترك

نحوّل الألوان، المسافات، الخطوط والحالات إلى Contract مشترك بين Penpot وCSS Variables وBlazor Components.

04 · AGENTIC OPERATIONS

الوكيل يفهم ويقترح ويعدّل

يمكن للوكيل تحليل الاتساق، تسمية الطبقات، إنشاء Variants، تجهيز Screens جديدة، وتوثيق قواعد الربط مع الكود.

05 · GOVERNANCE

كل تعديل له Policy وأثر

عمليات الكتابة تمر عبر DraftOnly، Preview، Validation وApproval، بينما تسجل كل خطوة في Agent Observatory.

06 · PRODUCT LEARNING

من تجربة واحدة إلى Platform Capability

نبدأ بشاشة Supplier Approval، ثم نستخرج Template قابلًا لإعادة الاستخدام في كل Modules بدل بناء مولد عام قبل إثبات الجودة.

ARCHITECTURE FIT

الربط مع ما بنيناه بالفعل

لا ننشئ Stack منفصلًا؛ نركّب Design Capability فوق مكونات Sky365 الحالية حتى لا يظهر مسار تنفيذ موازٍ خارج الحوكمة.

داخل Sky365

  • Agent Workbench: نقطة إصدار أوامر التصميم ومراجعة المسودة.
  • Shared Action Core: بوابة موحدة لأوامر القراءة والكتابة.
  • Semantic Dictionary: ربط مصطلحات الأعمال بالمكونات المرئية.
  • Policy Corpus: قواعد من يقرأ، يعدّل، يوافق وينشر.
  • Agent Observatory: تتبع كل Tool Call وDiff وقرار.

داخل Penpot

  • Pages: تنظيم Domains وWorkflows بدل شاشات عشوائية.
  • Components & Variants: حالات موحدة وقابلة لإعادة الاستخدام.
  • Design Tokens: عقد مشترك قابل للتحويل إلى Code Variables.
  • MCP Plugin: وصول الوكيل إلى الصفحة النشطة فقط.
  • Libraries: فصل Core Design System عن Experiments.

NON-NEGOTIABLE GUARDRAILS

قواعد الأمان قبل أي Automation

توثيق Penpot نفسه يحذر من أن الوكيل يستطيع تنفيذ عمليات كتابة وحذف وتغيير على الصفحة النشطة. لذلك لا نسمح بالوصول الحر.

G01Read-Only First

ابدأ دائمًا بـinspect وlist وanalyze قبل أي كتابة.

G02Describe Before Apply

الوكيل يشرح التعديلات المقترحة ويعرض Scope واضحًا.

G03Small Reversible Steps

لا توجد أوامر “Refactor everything” على Design System حي.

G04Dedicated Draft Page

كل تجربة تتم في Draft Page أو Branch Library مستقلة.

G05Token Policy

لا ينشئ الوكيل Token جديدًا إذا كان Alias قائمًا يؤدي الغرض.

G06Human Approval

التغييرات البصرية العامة لا تنتقل إلى Core دون Reviewer.

G07Secret Hygiene

MCP Key تعامل ككلمة مرور ولا تظهر في Screenshots أو Logs.

G08Evidence by Design

كل تعديل يسجل Before/After وسبب التغيير ونتيجة التحقق.

30-DAY PILOT

خطة تنفيذ صغيرة لكنها حاسمة

لن نبني “مولد واجهات عام” من البداية. سنثبت الجودة على Workflow واحد عميق، ثم نعمم الناتج.

PHASE 01 · DAYS 1–7

Foundation

  • تشغيل Penpot SaaS أو Self-Hosted Pilot.
  • إنشاء Sky365 Core Tokens.
  • تجهيز Component Naming Contract.
  • ربط MCP في وضع القراءة أولًا.
Exit gate: الوكيل يقرأ الملف ويخرج Inventory صحيحًا دون تعديل.
PHASE 02 · DAYS 8–18

Supplier Approval

  • تصميم شاشة Supplier Approval المرجعية.
  • تعريف الحالات والصلاحيات والـVariants.
  • إنشاء Draft عبر MCP في صفحة معزولة.
  • مراجعة بشرية وتثبيت Design Contract.
Exit gate: التصميم مقبول ويغطي الحالات الحرجة والـEmpty/Error states.
PHASE 03 · DAYS 19–30

Code & Evidence

  • تحويل التصميم إلى Blazor Components Typed.
  • Visual Diff + Accessibility + Component Tests.
  • تسجيل الرحلة في Flight Recorder.
  • استخراج Template وADR وقياسات الجودة.
Exit gate: PR قابل للمراجعة، وليس مجرد HTML مستخرج.

ARCHITECTURE DECISION

Penpot هو المحرك الأول، لا القفص الأخير.

نعتمد Penpot كتجربة أولى لأنه مفتوح المصدر، يدعم Self-Hosting، Design Tokens، وMCP متعدد الاتجاهات. لكن الـDomain الخاص بنا يظل Agentic Design Studio حتى نتمكن من استبدال الأداة أو إضافة Figma وCanva وأدوات أخرى دون تغيير جوهر المنصة.
